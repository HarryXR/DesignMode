# DesignMode
简述Java常用设计模式及设计原则

1. strate---------策略模式

   -  针对接口编程，而不是针对实现
   
   -  找出应用中可能需要变化之处，把它们独立出来，不要和那些不需要变化的代码混在一起。
   
   -  多用组合，少用继承
   
2. observer-------观察者模式

   - 为了交互对象之间的松耦合设计而努力
   
3. decorator------装饰者模式

   - 开放-关闭原则
              ：类应该对扩展开放，对修改关闭。
   
4. singleton------单例模式

   -  依赖抽象，不依赖具体类
